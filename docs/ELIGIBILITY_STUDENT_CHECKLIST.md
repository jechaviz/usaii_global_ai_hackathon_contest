# Student Eligibility Checklist

Use this before any external submission. Keep proof documents outside git.

## Team

- [ ] Team has 2-5 members.
- [ ] Every member registered individually on Devpost by 2026-06-06.
- [ ] Team track matches the highest education level represented.
- [ ] One student lead is named for Devpost project creation on 2026-06-14.
- [ ] All teammate invite emails match their Devpost registration emails.

## Student Status

- [ ] Each member is currently enrolled in an accredited secondary school,
  college, university, or graduate/doctoral program through 2026-06-21.
- [ ] High school students are grades 9-12 or covered by the official transition
  language.
- [ ] Undergraduate students meet the grade 13-16 / undergraduate status rule.
- [ ] Graduate students are enrolled in a Masters or PhD program by hackathon
  start.
- [ ] No member is an ineligible non-student.
- [ ] No member is disqualified by employment restrictions.

## Age And Consent

- [ ] Every member is at least 14 years old.
- [ ] Members under 18 have parent/guardian consent ready.
- [ ] Age verification is not committed to git.

## Qualifier

- [ ] Team completed qualifier between 2026-06-07 and 2026-06-10.
- [ ] Team received score report.
- [ ] Approval code received by email/platform by 2026-06-12.
- [ ] Approval code stored privately, not in repo.

## Prize Verification Readiness

- [ ] Enrollment proof ready if winner.
- [ ] Age verification and parental consent ready if applicable.
- [ ] Tax documentation plan ready if required.
- [ ] Winner agreement review owner named.

## Public Repo Boundary

- [ ] No school ID, birth date, tax ID, guardian email, home address, passport,
  account credential, or private student proof in public files.
- [ ] Public demo data is synthetic or explicitly consented.
- [ ] Final Devpost disclosures list tools, data sources, and AI assistance.
